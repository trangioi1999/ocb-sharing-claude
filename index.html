02 · Kiro có gì

Kiro — IDE có agent, của AWS

Yêu cầu→ requirements.md→ design.md→ tasks.md→ Code + Test
Specs

Mỗi tính năng 1 bộ 3 file: yêu cầu → thiết kế → danh sách việc. Mình duyệt từng bước rồi agent mới code.

Steering

File quy ước dự án (tech stack, cách đặt tên, cấu trúc) — agent tự đọc trước mọi câu trả lời.

Hooks

Canh sự kiện file (lưu / tạo / xóa) → tự chạy test, lint, cập nhật tài liệu.

2 kiểu làm việc

Spec session (đi đủ quy trình) và Vibe session (chat sửa code nhanh, bỏ qua spec).

2 mức giám sát

Autopilot (agent tự sửa nhiều file) và Supervised (duyệt từng lần sửa).

Nền tảng

IDE (nền VS Code) + CLI + Web + Mobile, chung 1 tài khoản. Có MCP + Powers (kho tích hợp).

02 · Kiro có gì · cấu trúc file thật

Kiro lưu "bộ não" trong repo thế nào

# Cây thư mục Kiro sinh ra .kiro/ ├─ specs/ │ └─ product-list-page/ │ ├─ requirements.md │ ├─ design.md │ └─ tasks.md ├─ steering/ │ ├─ product.md # sản phẩm là gì │ ├─ tech.md # stack, thư viện │ └─ structure.md # cấu trúc folder └─ hooks/ └─ on-save-run-test.json
# Ví dụ nội dung requirements.md (chuẩn EARS) ## Yêu cầu 2 — Lọc sản phẩm theo giá User Story: Là người mua, tôi muốn lọc sản phẩm theo khoảng giá. Acceptance Criteria: WHEN người dùng kéo thanh chọn giá THE SYSTEM SHALL cập nhật danh sách không tải lại trang IF không có sản phẩm nào khớp THEN THE SYSTEM SHALL hiện empty state kèm nút "Xóa bộ lọc" # tasks.md sinh tiếp: checklist từng việc, # bấm từng dòng cho agent làm → duyệt diff

Điểm hay: spec viết kiểu WHEN/SHALL nên mỗi dòng đều test được — QC lấy làm test case, dev lấy làm định nghĩa xong việc.

03 · Claude Code có gì

Claude Code — agent làm việc theo vòng lặp

Nhận việc→ đọc code liên quan→ sửa→ chạy test / build↺ lỗi thì tự sửa tiếp, xong mới dừng
CLAUDE.md

File quy ước dự án — giống Steering của Kiro. Agent đọc trước mọi việc.

Skills

"Gói bí kíp" cho từng loại việc (viết docs, xử lý Excel, deploy...) — chỉ nạp khi cần, đỡ tốn context.

Subagents

Agent con chạy song song, mỗi con một vùng nhớ riêng — việc to chia nhỏ làm cùng lúc.

Plan mode

Bắt agent trình kế hoạch, mình duyệt xong mới được sửa code — giống "spec nhẹ".

Hooks

Tự chạy lệnh trước/sau hành động của agent: sửa file xong thì lint, trước commit thì test.

MCP

Cổng chuẩn để agent gọi tool ngoài: Figma, GitHub, Playwright, Jira... Cài như cài extension.

Chạy trong terminal, có plugin hiển thị trong VS Code/JetBrains · chạy được không cần người (trong CI/CD) · không phụ thuộc cloud nào.

03 · Claude Code có gì · cấu trúc file thật

Claude Code lưu "bộ não" trong repo thế nào

# Cây thư mục trong dự án FE CLAUDE.md # quy ước, agent luôn đọc .claude/ ├─ skills/ │ └─ write-spec/ │ └─ SKILL.md # bí kíp viết spec ├─ agents/ │ └─ code-reviewer.md ├─ commands/ │ └─ fix-issue.md # lệnh tắt /fix-issue └─ settings.json # quyền + hooks .mcp.json # khai báo tool ngoài
# Ví dụ nội dung CLAUDE.md của dự án FE # Quy ước dự án - Stack: React 19 + TypeScript + Tailwind - Component đặt trong src/components/<domain>/ - Dùng shadcn/ui, KHÔNG tự viết button/modal mới - State: TanStack Query, không dùng Redux - Mọi component phải có test (Vitest + RTL) # Lệnh hay dùng - npm run dev · npm run test · npm run lint # Điều cấm - Không sửa file trong src/legacy/ - Không commit thẳng vào main

Cả hai bên đều là file Markdown nằm trong Git → viết một lần, cả team dùng chung, review được như code.

03 · Claude Code có gì · điểm mạnh lớn nhất

Đưa file gì vào cũng xử lý được

Word / Excel / PDF / PowerPoint

Anthropic phát hành bộ skills chính thức, mã nguồn mở (docx · xlsx · pdf · pptx): đọc, tạo, chỉnh sửa trực tiếp. BA gửi spec bằng Word, BU gửi data bằng Excel → đưa thẳng cho agent, không cần copy-paste tay.

HTML / CSV / Markdown / JSON / ảnh

Đọc trực tiếp không cần skill. Dán screenshot màn hình lỗi → agent nhìn ảnh, tìm code liên quan, sửa. Dán ảnh design → dựng component.

Prompt linh hoạt 3 kiểu

① Chat qua lại trong terminal · ② Một phát ăn ngay: claude -p "..." · ③ Nối ống kiểu Unix: cat loi.log | claude "vì sao?"

Phân tích = chạy code thật

Không "đọc rồi đoán" — agent viết script (Python/Node) chạy trên file thật rồi trả kết quả. Số liệu là số chạy ra, không phải số bịa.

# Ví dụ thật — BA gửi spec.docx, BU gửi danh sách field bằng Excel: $ claude -p "đọc spec.docx và fields.xlsx, đối chiếu với form hiện tại trong src/components/checkout/ — liệt kê field còn thiếu, field sai validation, xuất báo cáo ra gap-report.md"

03 · Claude Code có gì · không chỉ mình Claude Code

Hệ sinh thái Claude — 1 tài khoản, nhiều công cụ

Claude Code

Agent code trong terminal — nhân vật chính hôm nay.

Claude app + Artifacts

Vẽ nhanh mockup UI tương tác được (React/HTML) ngay trong chat — duyệt ý tưởng design với BA/BU trước khi code thật.

Cowork (desktop)

Agent làm việc với file trên máy: dọn tài liệu, tổng hợp báo cáo, xử lý hàng loạt — cho việc ngoài code.

Claude in Chrome

Agent thao tác trình duyệt thật — điền form, kiểm tra flow web đang chạy.

Connectors

Kho kết nối sẵn (Google Drive, Jira, Slack, GitHub, Figma...) — cài 1 click, agent đọc được dữ liệu team đang dùng.

Plugins + Agent SDK

Đóng gói skills + MCP thành plugin chia sẻ cả team; SDK để tự build agent riêng khi cần.

Kiro cũng có IDE + CLI + Web + Mobile, nhưng đều xoay quanh việc code. Hệ Claude phủ rộng hơn: code (Claude Code) + design nhanh (Artifacts) + việc văn phòng (Cowork) + browser (Chrome) — cùng 1 subscription.

04 · So sánh cho FE

Luồng chạy: spec → dev → test

Kiro
yêu cầu→ spec 3 file (duyệt từng bước)→ làm theo tasks.md→ hooks tự chạy test khi lưu→ duyệt diff trong IDE
Claude Code
yêu cầu→ plan mode (duyệt kế hoạch)→ agent code + tự chạy test↺ tự sửa đến khi pass→ commit / mở PR

Kiro: quy trình là mặc định, tool ép mình theo. Hợp khi muốn kỷ luật đến từ công cụ. Nhược: sửa cái nút cũng dễ bị kéo vào vòng spec.

Claude Code: quy trình là tùy chọn — việc to thì bật plan mode, việc nhỏ chạy thẳng. Kỷ luật do team tự ghi vào CLAUDE.md + skills.

Cho việc chuyển giao: luồng spec-trước-code của Kiro tái hiện được trên Claude Code (plan mode + skill viết spec) — không mất quy trình, chỉ đổi chỗ để cấu hình.

04 · So sánh cho FE · demo gợi ý

3 ví dụ chạy thật được ngay

# 3 · Bug chỉ tái hiện trên browser $ claude > trang /orders bị nhảy layout khi bấm filter. Mở browser tái hiện, chụp màn hình trước/sau, tìm nguyên nhân, sửa và chạy lại cho tôi xem # Playwright MCP: bấm, chụp, # đọc console → sửa → xác nhận ✓
